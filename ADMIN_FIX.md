# Решение проблемы с командой /admin

## Проблема
Если вы получаете сообщение "❌ Команда недоступна - Административная панель не настроена", это означает, что переменная окружения `ADMIN_USER_ID` не передается в процесс бота.

## Решение

### Вариант 1: Перезапуск бота через скрипт (рекомендуется)

1. **Остановите бота** (Ctrl+C в терминале)

2. **Запустите через скрипт:**
   ```powershell
   .\start.ps1
   ```

   Скрипт установит переменную окружения и запустит бота.

### Вариант 2: Установка переменной вручную

Если вы запускаете бота вручную (не через скрипт), установите переменную перед запуском:

```powershell
$env:ADMIN_USER_ID = "7217012505"
go run ./cmd/feedback-bot
```

### Вариант 3: Проверка в логах

При запуске бота проверьте логи. Должна быть строка:
```
{"level":"info","msg":"admin user configured","admin_user_id":7217012505}
```

Если вы видите:
```
{"level":"warn","msg":"admin user not configured",...}
```

То переменная не установлена.

## Как проверить, что переменная установлена

Перед запуском бота выполните:
```powershell
Write-Host "ADMIN_USER_ID = $env:ADMIN_USER_ID"
```

Если выводится пустота, переменная не установлена.

## Важно!

- Переменная окружения устанавливается **только для текущей сессии PowerShell**
- Если вы закрыли и открыли новый терминал, нужно снова установить переменную
- Скрипт `start.ps1` делает это автоматически при каждом запуске
